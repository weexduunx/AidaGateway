# Contributing

Merci de votre intérêt pour contribuer à Laravel Aida Gateway !

## Comment contribuer

1. Fork le projet
2. Créez une branche pour votre feature (`git checkout -b feature/AmazingFeature`)
3. Committez vos changements (`git commit -m 'Add some AmazingFeature'`)
4. Poussez vers la branche (`git push origin feature/AmazingFeature`)
5. Ouvrez une Pull Request

## Lignes directrices

### Code Style

- Suivez les standards de code PSR-12
- Utilisez les types strictes (`declare(strict_types=1)`)
- Documentez votre code avec des PHPDoc

### Tests

- Écrivez des tests pour toute nouvelle fonctionnalité
- Assurez-vous que tous les tests passent avant de soumettre une PR
- Maintenez une couverture de code élevée

```bash
composer test
```

### Documentation

- Mettez à jour le README.md si nécessaire
- Ajoutez des exemples d'utilisation
- Documentez les nouvelles fonctionnalités dans le CHANGELOG.md

### Commit Messages

Utilisez des messages de commit clairs et descriptifs :

- `feat: Add new gateway support`
- `fix: Resolve webhook signature validation`
- `docs: Update installation instructions`
- `test: Add tests for payment refund`

## Signaler des bugs

Si vous trouvez un bug, veuillez ouvrir une issue avec :

- Une description claire du problème
- Les étapes pour reproduire le bug
- Votre environnement (version PHP, Laravel, etc.)
- Le comportement attendu vs le comportement actuel

## Proposer des améliorations

Nous sommes ouverts aux suggestions ! Pour proposer une amélioration :

1. Ouvrez une issue pour discuter de votre idée
2. Attendez les retours avant de commencer à coder
3. Suivez le processus de contribution ci-dessus

## Code de conduite

Soyez respectueux et professionnel dans toutes vos interactions avec la communauté.

Merci de contribuer à rendre ce package meilleur !
